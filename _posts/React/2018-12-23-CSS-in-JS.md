Note on using CSS in JS

## #Flaws in CSS
- When using CSS for small applications in React we use CSS by namespacing it to a component element.
    - Styles leaking across components because of shared namespacing.
    - So in future, if another developer uses the same namespace for different elemenet. It will create problem with two element share same namespace.
    
- Solution: 
    - Block Element Modifier, BEM, naming convention.
    - CSS in JS using `styled-components`.

## #CSS in JS using `styled-components`
- CSS in JS is using a JavaScript library to render our style for a component.
    - Functional scoping, i.e., using `style` attribute of an element in react also solves the problem. But with this approach we don't have access to all of the selectors.
- `$ yarn add styled-components`
    - Use it to generate components.
    - It gives the component a unique string for class of the component.
    - In the `head` of the application, appends CSS for that unique class.
    - This prevents style bleeding.
- We can use props and all the CSS selectors using CSS in JS.

```js
import React from 'react';
import styled from 'styled-components';

const Text = styled.div`
    color: red;
    font-size: 28px;
    border: ${({ isActive }) => 
        isActive ? '1px solid black' : '3px dotted green'`};
`;

function App() {
    return (
        <div class='App'>
            <Card>
                <Text isActive={false}>I am a component</Text>
            </Card>
        </div>
    );
}
```

## #Sharing propeties between different elements:

```js
import styled, { css } from 'styled-components';
import { Link } from 'react-router-dom';

const OptionsContainerStyles = css`
    padding: 10px 15px;
    cursor: pointer;
`;

export const OptionLink = styled(Link)`
    ${OptionsContainerStyles}
`;

export const OptionDiv = styled.div`
    ${OptionsContainerStyles}
`;
```

- OR

```js
<OptionLink as='div' onClick={() => auth.signOut()}>
    SIGN OUT
</OptionLink>
```
